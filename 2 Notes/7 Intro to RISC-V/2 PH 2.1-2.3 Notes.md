2.2 introduced you to some basic operations in RISC, like addition.

```
add a, b, c // The sum of b and c is placed in a
```

And they gave a simple example of how C code would be translated into this assembly.

```c
a = b + c; 
d = a −e;
```

```
add a, b, c 
sub d, a, e
```

And how intermediate variables might be introduced if the operation becomes more complicated than just 2 variables.

```c
f = (g + h) −(i + j);
```

```c
add t0, g, h // temporary variable t0 contains g + h
add t1, i, j // temporary variable t1 contains i + j
sub f, t0, t1 // f gets t0 −t1, which is (g + h) −(i + j
```


2.3 talk about registers on a RISC-V computer, how a register stores 64 bits of information, and by default design, there are 32 registers on the RISC-V computer.

Most calculations, like the ones we've seen above only work with values that are stored inside the registers, not the memory, and so all those values must be loaded into the registers first from memory.

This introduces data transfer instructions, which transfers data between memory and register addresses.

![[PH - Computer Organization and Design RISC-V Edition The Hardware Software Interface (David A. Patterson, John L. Hennessy).jpg|500]]

[[PH - Computer Organization and Design RISC-V Edition The Hardware Software Interface (David A. Patterson, John L. Hennessy).pdf#page=176&rect=72,398,538,781|PH - Computer Organization and Design RISC-V Edition The Hardware Software Interface (David A. Patterson, John L. Hennessy), p.176]]

Take this example in C:

```c
g = h + A[8]
```

This would require a lot of data transferring between memory and registers:

```
la   x22, A          # x22 = base address of array A
ld   x21, h          # load h from memory into x21
ld   x9, 8(x22)     # load A[8] into x9
add  x20, x21, x9    # x20 = h + A[8]
sd   x20, g          # store result back to memory location g
```

The register added to form the address (x22) is called the base register, and the constant in a data transfer instruction (8) is called the offset.
