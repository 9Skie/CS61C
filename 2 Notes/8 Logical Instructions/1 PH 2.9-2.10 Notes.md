2.9

This is about how Chars are being stored (ah, we already know ASCII), and then goes on to show single bytes can be read/written to from a source or to a destination.

The reading/writing all only happens on the rightmost 8 bits of a register.

```
lbu x12, 0(x10) // Read byte from source 
sb x12, 0(x11) // Write byte to destination
```

They used this syntax to show an example of how C's strcpy (a function that copies a string into another string) runs in RISC-V assembly.

(I changed it a little because I don't understand the C syntax they had)

```c
void strcpy(char dest[], char src[]) {
    size_t i = 0;

    while (src[i] != '\0') {
        dest[i] = src[i];
        i++;
    }

    dest[i] = '\0';   // IMPORTANT: copy the terminator
}
```

In RISC-V:

```
# strcpy(dest=x10, src=x11)
# i in x19 (callee-saved)
# temps: x5, x6, x7

strcpy:
    addi sp, sp, -8          // make space to save x19
    sd   x19, 0(sp)          // save old x19
    add  x19, x0, x0         // size_t i = 0;

L1:
    add  x5, x11, x19        // x5 = &src[i]
    lbu  x6, 0(x5)           // x6 = src[i]  (unsigned byte)

    beq  x6, x0, Ldone       // while (src[i] != '\0') ... ; if src[i]==0, break

    add  x7, x10, x19        # x7 = &dest[i]
    sb   x6, 0(x7)           # dest[i] = src[i]

    addi x19, x19, 1         # i++
    jal  x0, L1              # loop

Ldone:
    add  x7, x10, x19        # &dest[i]
    sb   x0, 0(x7)           # dest[i] = '\0'   (copy terminator)

    ld   x19, 0(sp)          # restore old x19
    addi sp, sp, 8           # pop stack
    jalr x0, 0(x1)           # return
```

So, obviously... this adds quite a bit of confusion, since we have a lot more instructions.

They do introduce these instructions in the lectures, so no worries.
